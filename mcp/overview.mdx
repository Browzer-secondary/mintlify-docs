Browzer's MCP server gives AI agents direct, structured access to your workflow documentation. Instead of copy-pasting SOPs into a chat window or maintaining a separate knowledge base, your AI tools can query your live Browzer workflows in real time — so the answers they give your team are always grounded in the process documentation you've already built.

## What is the Model Context Protocol?

The Model Context Protocol (MCP) is an open standard that defines how AI agents connect to external data sources and tools. Think of it as a universal adapter: any MCP-compatible client — whether that's Claude, Cursor, or a custom agent you build yourself — can connect to any MCP server using the same protocol, without custom integration work on either side.

By running an MCP server, Browzer becomes a first-class data source for any AI agent that speaks the protocol. Your documented workflows stop being static files and become a live, queryable knowledge layer.

## What Browzer's MCP Server Does

Browzer's MCP server exposes the workflows stored in your [Knowledge Graph](/workflows/knowledge-graph) as structured, searchable context. Once your MCP client is connected and authenticated, it can retrieve workflow steps, search across your SOPs, and surface answers drawn directly from the documentation your team has recorded and edited.

The server is read-only from the agent's perspective — it retrieves your documentation but does not modify it. Every request is authenticated with your Browzer API key, so only the clients you authorize can access your content.

## Use Cases

You can use Browzer's MCP server to build or enhance a wide range of AI-powered tools:

- **Team process Q&A** — Connect an AI assistant to your MCP server so teammates can ask questions like "How do I submit a PTO request?" and get step-by-step answers pulled from your actual Browzer workflows.
- **Onboarding acceleration** — Give new hires an AI onboarding companion that answers questions using the documented SOPs in your Knowledge Graph, reducing the time senior staff spend on repetitive explanations.
- **AI tools with workflow context** — Build custom agents or automations that use your process documentation as grounding context, ensuring outputs stay consistent with how your team actually works.
- **Documentation search** — Let AI clients surface the right workflow for a task by searching across your entire Knowledge Graph rather than relying on users knowing exact document titles.

## What You Need to Get Started

Before connecting an MCP client to Browzer, make sure you have:

- An active Browzer account
- At least one workflow published in your [Knowledge Graph](/workflows/knowledge-graph)
- A Browzer API key (used to authenticate your MCP client)

If you haven't set up your Knowledge Graph yet, start with [editing your workflow documentation](/workflows/edit-workflow-documentation) and then return here once you have workflows ready.

## Authentication

All requests to Browzer's MCP server are authenticated using a Browzer API key passed as a Bearer token in the `Authorization` header. You generate this key once inside Browzer's settings, then add it to your MCP client's configuration. Keep the key secret — anyone who holds it can query your workflow documentation.

See [Create an API Key](/mcp/create-api-key) for the full walkthrough.
